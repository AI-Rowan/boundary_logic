============================================================
SUMMARY: 00_pima_external_test.R
External test data workflow — prediction-colour biplot
============================================================

PURPOSE
-------
Demonstrates how to produce a biplot for test data that was
never passed to bl_prepare_data(), and whose true labels are
unknown. Points are coloured by predicted class only:
  Red  = predicted class 1 (diabetic)
  Blue = predicted class 0 (non-diabetic)

No package functions were changed. The only differences from
the standard workflow are in how the data is prepared and how
the test data is passed to bl_project_points().


R FILES USED (boundary_logic package)
--------------------------------------
R/data_prepare.R       bl_prepare_data()
                         Encodes the class column as 0/1,
                         selects feature columns, and splits
                         into train/test sets.

R/outlier_filter.R     bl_filter_outliers()
                         Removes outlier rows based on a
                         convex-hull fraction filter.

R/model_utils.R        bl_wrap_model()
                         Wraps an externally fitted model so
                         the rest of the pipeline can score
                         new data through it.

R/projection.R         bl_build_projection()
                         Computes the projection matrix V
                         (PCA or CVA) and stores centering/
                         scaling metadata.

R/biplot_grid.R        bl_build_grid()
                         Creates a dense prediction grid in
                         Z-space used to render the coloured
                         background and decision boundary.

R/result.R             bl_assemble()
                         Combines all Phase 1 outputs into a
                         single bl_result object.

R/project_points.R     bl_project_points()
                         Projects any data frame into Z-space
                         using the stored V matrix and scores
                         each row through the fitted model.
                         If no "class" column is present,
                         colours points red (predicted 1) or
                         blue (predicted 0) automatically.

R/plot_biplot.R        plot_biplotEZ()
                         Renders the biplot: background grid,
                         data points, variable axes, and
                         decision boundary contour.


STEPS FOLLOWED
--------------
Step 1 — Manual train/test split (BEFORE the pipeline)
  The full Pima dataset is split outside the package:
    - 80% kept as labelled training rows (train_raw)
    - 20% held out as external test rows (test_raw)
  This split is done with base R sample(), not by the package.

Step 2 — Prepare training data only
  bl_prepare_data() is called on train_raw with
  train_fraction = 1, so all labelled rows become training
  data. No internal test split is made. The key difference
  from the standard workflow is that test_raw never enters
  bl_prepare_data() at all.

Step 3 — Filter outliers (optional)
  bl_filter_outliers() removes outlier rows from the training
  set using a convex-hull fraction filter.

Step 4 — Fit and wrap a custom GAM
  mgcv::gam() is fitted directly on the filtered training
  data with spline terms for Glucose and BloodPressure.
  bl_wrap_model() wraps the fitted model so it integrates
  with the rest of the pipeline.

Step 5 — Build CVA projection
  bl_build_projection() computes the CVA projection matrix V,
  centering vector X_center, and scaling metadata. bl_model
  is supplied so confusion labels (TP/TN/FP/FN) can be
  derived, which CVA requires to have >= 3 class levels.

Step 6 — Build prediction grid
  bl_build_grid() creates the dense Z-space grid used for
  the coloured background and the decision boundary contour.

Step 7 — Assemble result object
  bl_assemble() packages all Phase 1 outputs into a single
  bl_result object for use in plotting.

Step 8a — Reference biplot (training data, confusion colours)
  plot_biplotEZ(result) is called with no points argument.
  The function defaults to projecting result$train_data,
  which has a "class" column, so confusion colours are used:
    TP = red, TN = blue, FP = purple, FN = orange.

Step 8b — External test biplot (prediction colours only)
  The Outcome column is dropped from test_raw, leaving only
  the feature columns. bl_project_points() detects no "class"
  column and assigns red/blue by predicted class. The result
  is passed to plot_biplotEZ() via the points argument.

Step 8c — Optional sanity check (true labels reattached)
  The Outcome column is renamed to "class" and reattached
  to the test features. bl_project_points() now sees a
  "class" column and assigns confusion colours, allowing the
  user to inspect where the model made errors. This step is
  purely diagnostic and would be skipped in a real scenario
  where labels are genuinely unknown.


KEY INSIGHT
-----------
The colour-switching behaviour is entirely controlled by
whether the data frame passed to bl_project_points() contains
a column named "class". Dropping that column (Step 8b) is
the single change that produces prediction-only colours.
No package functions need to be modified.
